# Contributing to Navixy Dashboard Documentation

Thank you for your interest in contributing to this documentation project! This guide will help you get started with contributing.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or request features
- Provide clear descriptions and steps to reproduce
- Include relevant screenshots or error messages
- Check existing issues before creating new ones

### Suggesting Improvements
- Use GitHub Issues for documentation improvements
- Provide specific suggestions for better content
- Explain the benefit of your proposed changes

### Contributing Content

#### Documentation Updates
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test your changes locally
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

#### Adding Examples
1. Follow the existing structure in `/docs/examples/`
2. Include clear descriptions and use cases
3. Provide configuration files when applicable
4. Test all examples before submitting

#### Creating Templates
1. Follow the existing structure in `/docs/templates/`
2. Include comprehensive documentation
3. Provide JSON configuration files
4. Test templates in a clean dashboard instance

## Content Guidelines

### Writing Style
- Use clear, concise language
- Write for your target audience
- Include practical examples
- Provide step-by-step instructions

### Documentation Structure
- Follow the existing directory structure
- Use consistent naming conventions
- Include table of contents for long documents
- Cross-reference related content

### Code Examples
- Use proper syntax highlighting
- Include comments explaining complex parts
- Test all code examples
- Provide context for configuration values

## Review Process

### Pull Request Requirements
- Clear description of changes
- Reference to related issues
- Screenshots for UI changes
- Testing instructions

### Review Criteria
- Content accuracy and clarity
- Adherence to style guidelines
- Proper formatting and structure
- Completeness of examples

## Development Setup

### Prerequisites
- Git
- Text editor with Markdown support
- Dashboard runtime for testing

### Local Development
1. Clone the repository:
   ```bash
   git clone https://github.com/DanilNezhdanov/navixy-dashboard-schema.git
   ```
2. Create a new branch
3. Make your changes
4. Test locally
5. Submit a pull request

## Code of Conduct

### Our Pledge
We are committed to providing a welcoming and inclusive environment for all contributors.

### Expected Behavior
- Use welcoming and inclusive language
- Be respectful of differing viewpoints
- Accept constructive criticism gracefully
- Focus on what's best for the community

### Unacceptable Behavior
- Harassment or discrimination
- Trolling or inflammatory comments
- Personal attacks
- Inappropriate or offensive content

## Recognition

Contributors will be recognized in:
- CONTRIBUTORS.md file
- Release notes for significant contributions
- GitHub contributor statistics

## Questions?

If you have questions about contributing:
- Open a GitHub Issue
- Check existing documentation
- Review closed issues for similar questions
